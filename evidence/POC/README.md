# POC Evidence Directory

## Purpose

This directory stores evidence artifacts generated during the Proof of Concept phase.

## Structure

```
POC/
├── README.md           # This file
├── week-0/             # Week 0 evidence
│   ├── role-assignments/
│   ├── kickoff/
│   └── timeline/
├── week-1/             # Week 1 evidence
│   ├── a3-outputs/
│   ├── a4-outputs/
│   ├── adjudication/
│   └── ci-runs/
└── week-2/             # Week 2 evidence
    ├── integration/
    ├── validation/
    └── final-report/
```

## Evidence Types

### Role Assignments
- Acknowledgment screenshots/logs
- Assignment issue links
- Capacity confirmations

### CI Runs
- Workflow run logs
- Test results
- Frequency reports

### Adjudication
- A3 vs A4 comparison results
- Verdict records
- Diff analysis

### Validation
- Thunder Strike reports
- Self-assessments
- External validation records

## Naming Convention

```
{type}_{date}_{sequence}.{ext}

Examples:
- role_assignment_20240115_001.md
- ci_run_20240116_042.json
- adjudication_20240117_001.yaml
```

## Retention

- POC evidence retained for 90 days minimum
- Critical decisions retained indefinitely
- Linked to A7 bundles for audit trail
