# A7 Evidence Bundle Directory

This directory contains evidence bundles generated by `compute_f_total.py`.

## Structure

```
A7-bundle/
├── README.md                          # This file
└── evidence_bundle_YYYYMMDD_HHMMSS/   # Timestamped bundles
    ├── frequency_report.json          # Machine-readable report
    ├── frequency_report.yaml          # Human-friendly report
    └── SUMMARY.md                     # Executive summary
```

## Generating a Bundle

```bash
# With demo data
python scripts/compute_f_total.py --demo -o evidence/A7-bundle

# With manifest configuration
python scripts/compute_f_total.py -m manifest.yaml -o evidence/A7-bundle

# JSON output only
python scripts/compute_f_total.py --demo --json
```

## Bundle Contents

### frequency_report.json
Machine-readable frequency report containing:
- Timestamp
- F_total score
- Frequency band classification
- Component breakdown
- Validation records
- Diamond Hands readiness flag
- Recommendation

### frequency_report.yaml
Human-readable version of the same data.

### SUMMARY.md
Executive summary with:
- Key metrics at a glance
- Component frequency table
- Validation pass count
- Actionable recommendation

## Retention Policy

- Bundles are retained for **90 days** minimum
- Critical decision bundles retained indefinitely
- CI/CD uploads bundles as artifacts with 90-day retention

## Usage in Validation

Thunder Strike validators reference evidence bundles during:
1. Self-Assessment (Level 1) - Self-generated bundle
2. Peer Review (Level 2) - Verify peer's bundle
3. External Validation (Level 3) - Independent bundle verification
4. Diamond Hands Final (Level 4) - Complete bundle review

## Artifacts Uploaded by CI/CD

GitHub Actions uploads bundles as:
- `evidence-bundle-{run_number}`
- Available in workflow artifacts
- Downloadable for 90 days
